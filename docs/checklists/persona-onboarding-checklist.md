# Persona Onboarding Checklist

Use this checklist when assigning a named person to a role for a project.

- [ ] Role name and assigned person added to project README
- [ ] Role responsibilities summarized in the project README or role section
- [ ] Key contacts and communication channels documented (Slack, email, meetings)
- [ ] Expected deliverables and handoff points listed (e.g., design assets, architecture docs, runbooks)
- [ ] Access required (repos, dashboards, CI/CD, staging) requested and granted
- [ ] Knowledge transfer session scheduled (if needed)
- [ ] Acceptance criteria and success metrics agreed with Product/PM
- [ ] Link to related templates (release checklist, risk register, runbooks)
